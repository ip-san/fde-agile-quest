# narrative-designer — 総監督ノート

作業前に必読。総監督から受けた「言い回し・トーン・テンポ・キャラの声・避けるべき定型句」の観点を、
毎回守る再発防止ルールとして扱う。

## 2026-06-20
- [バックログ連動] イベント本文を、その時点のバックログ状態（forecast に積んだ PBI・着手中・Done・未完／キャリーオーバー）に呼応させて書く。daily は今スプリントで追っている PBI を名指しで進捗に触れ、review／retro は「実際に Done した項目」と「持ち越した項目」を本文で受ける。バックログに一度も触れず、セレモニーを順番に消化するだけの本文にしない。（背景: 総監督「バックログの消化が話に組み込めていない」。現状 events-sprint*.ts は backlogDone/carryover をほぼ参照せず物語と別レールで走っている）
- [未完を引き受ける] 完了していない項目があるのに、物語だけ「片付いた」体で次へ進める書き方をしない。未消化・持ち越しは次スプリントの planning／daily の入りで物語が引き取る（例: before=未完に触れず新スプリントの新ネタへ → after=「結局◯◯は手つかずのまま持ち越した」と前段の未完を受けてから進む）。（背景: 総監督「完成してないのに次々話が進む」）
