# ux-engineer — 総監督ノート

作業前に必読。総監督から受けた「操作感・余白・モーション・モバイルの当たり判定・配色」の観点を、
毎回守る再発防止ルールとして扱う。

## 2026-06-20
- [操作の手応え] ON/OFF・押下・確定など状態が変わる操作は、**視覚（色＋動き）・聴覚（効果音）・
  触覚（押下感）の3層**でフィードバックを返す。色の変化だけで済ませない。
  （背景: 総監督「チェックボックスにチェックした感覚が少なく分かりにくい」。グリフ☐→☑の色変化のみで、
   音もアニメも押下感も無く手応えが弱かった）
  （例: before=`bg-sky-500/15`の色変化だけ → after=`sfxTick`の効果音＋`check-pop`でグリフが弾む
   ＋`active:scale-[0.98]`の押下感＋`ring`で選択状態を強調。モーション過敏は `prefers-reduced-motion` で無効化）
