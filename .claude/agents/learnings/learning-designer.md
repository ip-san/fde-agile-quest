# learning-designer — 総監督ノート

作業前に必読。総監督から受けた教育監修の観点を、毎回守る再発防止ルールとして扱う。

## 2026-06-20
- [完了の重み] 「未完のまま物語が進む」ことを当然としない設計監修をする。バックログ項目の未消化・持ち越しが、信頼／文化メーターや手戻りイベントとして体感できる結果に返るよう点検する。DoD・WIP 規律（＝「Done とは何か」を学ぶ）がこのゲームの核なので、完了を伴わない進行を許すと学びが空洞化する。物語の各スプリント遷移で「やり残しに見合う代償が描かれているか」を必読観点にする。（背景: 総監督「完成してないのに次々話が進む／バックログ消化が話に組み込めていない」）
- [核活動を必須フローに織り込む] このゲームの学びの核＝開発（着手→レビュー→Done）が、日課の“横の小窓”に切り出され、開かなくても日が自動で終わる構造になっていないかを点検する。デイリーは「現場の出来事を1つ解決すると自動で次へ進む（chooseCore→advanceCore）」ため、開発に一度も触れずスプリントが終わりうる＝学習対象を素通りできてしまう。学ばせたい活動は“やらないと進めない”か、少なくとも“やっていないことが毎日はっきり見える”形でメインの流れに置く。新しい日課・セレモニー設計をレビューするときは「核となる学習行動が、任意の脇道ではなく流れの中に組み込まれているか」を必ず問う。（背景: 総監督「その日が自動で終わるせいで、開発に手をつけず次に進む構造になっている」）
